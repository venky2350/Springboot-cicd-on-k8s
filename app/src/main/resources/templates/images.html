CI/CD Process Explained:
  • Step 1: Developer pushes code to GitHub.
  • Step 2: GitHub triggers a Jenkins build via webhook.
  • Step 3: Jenkins builds the application using mvn clean package.
  • Step 4: Jenkins performs code analysis using SonarQube and generates a coverage report.
  • Step 5: Jenkins uses Trivy to scan the Docker image for vulnerabilities (CVEs).
  • Step 6: After a successful scan, the image is pushed to DockerHub.
  • Step 7: Argo CD auto-syncs the change and upgrades the app using Helm.
  • Step 8: Kubernetes rolls out the new version of the application.
  • Step 9: On success or failure, the team receives a notification via Slack.
